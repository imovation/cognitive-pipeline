# Skill: 概念工程化器 (Concept Engineer)

## 0. 思想基石（Dependency Injection）
在开始工程化之前，请首先静默读取你自带的宪法：`references/Recursive_Abstraction_Model.md`，对齐如何顺应宿主生态物理法则与系统解耦的核心思想。

## 1. 核心使命 (Core Mission)
当你拿到一段纯理论或思想模型时，你的任务是将其转化为当前 Agent 系统（`opencode`）中真正可执行的底层基础设施（Skill、工具、SOP）。你只负责“画图纸（拆解理论）”，必须**调用官方的 `skill-creator` 工具**去干“砌砖（写文件）”的活。

## 2. 第一阶段：理论落地拆解 (Theoretical Implementation Decomposition)
请遵循以下 SOP，将理论拆解为具体可落地的工程结构暂存：

**第一步（对齐核心真理 - 道）：** 提取该理论的第一性原理。它触及了什么计算机科学或系统论底座？这决定了你要如何把它揉进提示词中，避免瞎猜忌。
**第二步（物理架构拆分）：**
- **大脑（SKILL.md 主体 - 法）：** 将理论的推导逻辑写成一步步的 SOP 或框架指引，作为 Agent 遇到类似情况时的强制思考回路。
- **四肢（自动化脚本 - 术）：** 识别出理论中可以自动化的部分，剥离成独立的命令行或小工具脚本。
- **试金石（Evals - 验证）：** 构思 2-3 个泛化的、现实世界可能会用到的测试提示词。

## 3. 第二阶段：生态链调用与移交 (Handoff to Builder)
完成物理拆分后，为你抽象出的新技能构思一个简短的英文技能名（如 `pareto-optimizer`）和一个高度概括其触发条件的描述（YAML description）。

接下来，**必须执行以下流水线交接操作：**
1. **加载 `skill-creator`：** 使用你的 `skill` 工具加载系统自带的官方 `skill-creator`。
2. **委托建造：** 将你拆分好的图纸（包括：第一性原理、SOP、四肢脚本、试金石），以及“侵略性的触发条件”，完整地喂给 `skill-creator`。
3. **遵循原生规范：** 听从 `skill-creator` 的引导（它会问你意图、让你确认用例、帮你写 SKILL.md、生成 evals 测试并物理落盘），直至新技能被官方引擎完美封装。

## 执行完成 (Completion)
向用户报告：“【演绎】物理拆解完毕，图纸已成功交接给官方 `skill-creator` 落地为底层基础设施，随时准备降维打击。”
